# LinkedIn Launch Posts

Launch posts announce product features, milestones, and releases. They convert followers into users and signal momentum.

---

## Format 1: Single Feature Launch

**When to use:** Shipping one feature or capability that has a clear user benefit.

**Structure:**
```
[Problem this solves - stated from the user's POV]

[What you built and how it works]

[Specific result or before/after]

[CTA: try it, link in comments]
```

**Template:**
```
[Audience] spend [time/effort] doing [painful task].

We just shipped [feature name] to fix that.

Here's how it works:
- [Step 1 / capability 1]
- [Step 2 / capability 2]
- [Step 3 / capability 3]

Early results: [specific metric or user quote].

Try it now - link in comments.

#product #category #launch
```

**Notes:**
- Lead with the problem, not the solution
- "We shipped" is better than "We're excited to announce"
- Include a specific result - a number, a time saved, a user quote
- If you have a demo GIF or screenshot, mention it in draft notes
- Link always goes in comments for better reach

## Format 2: Milestone Post

**When to use:** Hitting a round number (users, revenue, customers, months) that signals traction.

**Structure:**
```
[The number + what it means]

[The story behind getting there - what was hard]

[What you learned along the way]

[Thank the people who helped + what's next]
```

**Template:**
```
[Number] [users/customers/months/deployments].

[One sentence on what this means to you]

Getting here was not straightforward. [2-3 sentences on the real story - what almost went wrong, what you had to change, what surprised you]

What I learned:
- [Lesson 1 - specific and non-obvious]
- [Lesson 2]
- [Lesson 3]

Thank you to [people, early users, team]. Next milestone: [what you're aiming for].

#milestone #startup #category
```

**Notes:**
- The number alone is not interesting. The story behind it is.
- Be honest about what was hard - vulnerability drives engagement
- Thank specific people or groups
- Never inflate numbers or use vanity metrics without context

## Format 3: Launch Cluster (Multi-Feature or Launch Week)

**When to use:** Shipping multiple features at once, running a launch week, or doing a major version release.

**Structure:**
```
[The theme: what problem set you're addressing]

[List of what shipped - brief, punchy]

[The connecting thread - why these together, why now]

[CTA: deep dive link, demo, or try it]
```

**Template:**
```
We shipped [X] things this week. Here's the rundown:

1. [Feature]: [one-line benefit]
2. [Feature]: [one-line benefit]
3. [Feature]: [one-line benefit]
4. [Feature]: [one-line benefit]

The thread connecting all of these: [theme or user problem].

[One paragraph on the vision - where this is heading]

Full breakdown on the blog. Link in comments.

#launch #product #category
```

**Notes:**
- Number the items for scanability
- Each item gets exactly one line - save detail for the blog post
- The connecting thread is the most important part. Without it, it reads like a changelog.
- Works well paired with a blog post or video for the deep dive
- For launch weeks, consider one post per day with a final roundup
