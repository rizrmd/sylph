# Hemingway

Punchy writing rules for cutting bloat and sharpening prose. Apply to any content before it ships.

## The 8 rules

### 1. Cut the first paragraph
Your real opening is buried in paragraph two. Delete everything before it.

### 2. One idea per sentence
If a sentence has "and" connecting two ideas, split it. If it has a semicolon, split it. Each sentence earns its own period.

### 3. Replace adjectives with numbers
"Significant growth" means nothing. "3x growth in 6 months" means everything. If you can't find the number, cut the adjective.

### 4. Kill adverbs
"Very", "really", "extremely", "significantly", "actually", "basically", "essentially" - delete them all. If the verb needs an adverb, you picked the wrong verb.

### 5. Active voice only
"The report was generated" becomes "We generated the report." Find the actor. Make them the subject. If there's no actor, the sentence might not need to exist.

### 6. No weasel words
"Some people think", "it could be argued", "there is a sense that" - these are cowardice. State the claim or don't.

### 7. Read it aloud
If you stumble, the reader will too. If you run out of breath, the sentence is too long. If it sounds like a textbook, rewrite it.

### 8. Word count limits
- LinkedIn post: 150-300 words
- Email: under 200 words
- Blog intro: under 100 words
- Slack message: under 50 words
- Subject line: under 10 words

## Anti-patterns to catch

| Pattern | Fix |
|---------|-----|
| "In order to" | "To" |
| "Due to the fact that" | "Because" |
| "At this point in time" | "Now" |
| "It is important to note that" | Delete entirely |
| "Going forward" | Delete or use a date |
| "Leverage" (as verb) | "Use" |
| "Utilize" | "Use" |
| "Facilitate" | "Help" or "Run" |
| "Synergy" | Say what actually happens |
| "Ecosystem" | Be specific |
| "End-to-end" | Describe the actual scope |
| "Best-in-class" | Prove it with a number |
| "Cutting-edge" | Name the technology |
| "Seamless" | Describe the experience |
| "Robust" | Name the property (fast, reliable, tested) |
| "Innovative" | Show the innovation |

## Process

1. **Write the draft** - get it all out, don't self-edit yet
2. **Apply rules 1-6** - mechanical pass, cut 30-50% of word count
3. **Read aloud** (rule 7) - rewrite anything that sounds wrong
4. **Check word count** (rule 8) - cut again if over limit
5. **Final check** - scan for anti-patterns table above

## When to use

Apply Hemingway rules to:
- Any content in `_drafts/` before it moves to review
- Email drafts before sending
- Slack messages over 2 sentences
- PR descriptions and issue bodies
- Any customer-facing text

Do NOT apply to:
- Technical documentation (precision matters more than brevity)
- Legal or compliance text
- Direct quotes from others
