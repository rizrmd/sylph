# Blog - Channel Insights

> This file is the learning log for blog. Update it after every published piece.
> Skills read this file before generating new content to improve over time.

## Voice

- Direct, technical, founder-led. Short sentences. Fragments OK.
- Position as a humble, reliable tool — never hype.
- Always lowercase "pnv", even at start of sentence. Origin: "ponovo" = Croatian for "replay/again".
- No AI positioning. We are deterministic. Emphasize the difference.

## What works

- Evidence-based posts with real comparisons outperform abstract positioning
- Leading with the AI-agent blind spot (non-deterministic behavior) — this is the real pain point
- Short paragraphs, scannable structure, bold claims backed by specifics
- Comparisons against competitors (AI vs deterministic)
- How-it-works sections with numbered steps

## What doesn't work

- Marketing speak, superlatives, hype
- Positioning as "AI-powered" (we are the opposite)
- Long intros, weak openers ("we're excited to...")
- Framing a tradeoff that doesn't exist (e.g., "see vs cloud" — competitors like Browserbase already offer live viewing)

## Article types

- Product Update / Introduction — for new features or launches
- Comparison — positioning against competitors (Browserbase, Browser Use, etc.)
- How-To Guide — step-by-step use of cloud browser and replayable tasks
- Engineering — technical deep dives into how the cloud browser works

## Cadence

- No established cadence yet. Start with launch/intro post, then 1-2 posts/month.

## Audience

- Developers, QA engineers, ops teams who need browser automation
- They've tried Browserbase, Browser Use, Playwright, or Selenium
- They're skeptical of AI-driven browser automation (unreliable, non-deterministic)

## Distribution

- Cross-promote on LinkedIn and X/Twitter after publishing
- Each blog post should produce 1-2 social posts
